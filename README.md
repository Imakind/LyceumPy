# 🎓 Aqbobek Neural - Единый AI-портал Лицея

Единая цифровая экосистема для лицея Aqbobek, объединяющая академические данные, умное расписание (Smart Scheduler), AI-аналитику успеваемости и современный интерактивный интерфейс.

Проект разработан в рамках хакатона **AIS Hack 3.0** (Кейс 1: Aqbobek Lyceum).

## 🚀 Основные возможности
1. **AI-Ассистент и Мониторинг риска:** Предсказание вероятности завала СОЧ/СОР (Predictive Score) на основе трендов оценок и активности.
2. **Экстренные AI-замены (Emergency Protocol):** Автоматическое перестроение расписания при болезни учителя (алгоритм Smart Scheduler).
3. **Киоск-режим (Wall Newspaper):** Специальный эндпоинт для школьных экранов, выводящий топ учеников и анонсы.
4. **Mock BilimClass API:** Эмуляция внешней системы электронного дневника для тестирования интеграций.
5. **Нейросетевой UI:** Современный фронтенд на React/Next.js с использованием glassmorphism, темной темы и плавной анимации.

---

## 🛠 Стек технологий

* **Frontend:** Next.js (React), Tailwind CSS, Recharts (AreaChart), Lucide Icons
* **Backend:** FastAPI, SQLAlchemy (PostgreSQL), Pydantic v2
* **Infrastructure:** Docker, Docker Compose

## 📖 Инструкция по локальному запуску и тестированию

Проект полностью контейнеризирован. Самый простой способ запустить всю связку (База данных + Бэкенд + Фронтенд) — использовать Docker.

### Способ 1: Запуск через Docker Compose (Рекомендуемый)

1. **Клонируйте репозиторий и перейдите в папку проекта:**
   ```bash
   cd aqbobek-lyceum-portal
   ```

2. **Запустите контейнеры:**
   ```bash
   docker-compose up --build
   ```

3. **Откройте сервисы в браузере:**
   * **Frontend (UI):** [http://localhost:3030](http://localhost:3030)
   * **Backend Swagger (API Docs):** [http://localhost:8080/docs](http://localhost:8080/docs)

*(Примечание: в docker-compose порты проброшены как 3030 для фронта и 8080 для бэка, чтобы не конфликтовать с локальными сервисами).*

---

### Способ 2: Ручной запуск (Local Development)

Если вы хотите запустить проект локально без Docker (например, для разработки):

#### 1. Запуск Backend (FastAPI)
В корневой папке проекта:
```bash
# Создайте виртуальное окружение (опционально)
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate

# Установите зависимости
pip install fastapi uvicorn sqlalchemy pydantic psycopg2-binary

# Запустите сервер (по умолчанию порт 8000)
uvicorn main:app --reload
```
*API будет доступно по адресу `http://localhost:8000/docs`*

#### 2. Запуск Frontend (Next.js)
Откройте **новый** терминал:
```bash
cd frontend

# Установите зависимости
npm install

# Укажите адрес локального бэкенда
# (Если бэкенд запущен локально на 8000 порту)
export NEXT_PUBLIC_API_URL=http://localhost:8000 # Для Windows: set NEXT_PUBLIC_API_URL=http://localhost:8000

# Запустите сервер разработчика
npm run dev
```
*Фронтенд будет доступен по адресу `http://localhost:3000`*

## 🧪 Инструкция по тестированию интерфейса

После успешного запуска фронтенда откройте приложение в браузере. Вы увидите единый дашборд с переключением ролей в правом верхнем углу.

1. **Роль "STUDENT" (По умолчанию)**
   * Оцените интерактивный график `AreaChart` (Нейро-аналитика успеваемости). Наведите курсор на точки графика для проверки `Tooltip`.
   * В боковой панели работает блок **AI-Ассистент** с подсчетом вероятности завала СОЧ (15%).

2. **Роль "TEACHER"**
   * Кликните на кнопку `TEACHER` в меню сверху.
   * На главной панели появится дополнительный блок **AI-мониторинг риска**. 
   * Здесь ИИ-модель помечает учеников тегами `HIGH RISK` и `MEDIUM RISK` на основе падения активности или паттернов пропусков.

3. **Роль "ADMIN"**
   * Кликните на кнопку `ADMIN`.
   * В боковой панели появится красная кнопка **Emergency Protocol (Болезнь учителя)**. 
   * При клике на неё срабатывает (на уровне UI) вызов экстренного протокола перестроения расписания (Mock-алерт), в реальности на бэкенде вызывается метод POST `/api/admin/teacher-sick/{teacher_id}`.

## 📡 Эндпоинты API (Основные)

* `GET /api/analytics/predict/{student_id}` — Получить предсказание от AI для ученика.
* `GET /api/kiosk/wall-newspaper` — Получить данные для киоска (лидерборд, замены).
* `GET /api/mock/bilimclass/grades/{student_id}` — Эмуляция получения оценок из BilimClass.
* `POST /api/admin/teacher-sick/{teacher_id}` — Триггер для автоматического поиска замены учителя.
