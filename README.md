# 🎓 Aqbobek Lyceum School Portal Backend

Бэкенд-система для единого школьного портала, интегрирующая академические данные, умное расписание и ИИ-аналитику.

## 🛠 Стек технологий
* **Framework:** FastAPI
* **ORM:** SQLAlchemy (PostgreSQL)
* **Validation:** Pydantic v2
* **Background Tasks:** Встроенные средства FastAPI/Python для триггеров замен.
* **AI:** Предиктивные модели на основе сигмоидальных функций + Stub для LLM (OpenAI/LangChain).

## 📂 Структура проекта
* `models.py` — Описание таблиц БД (Пользователи, Оценки, Расписание, Ограничения).
* `schemas.py` — Pydantic-схемы для валидации запросов и ответов.
* `ai_service.py` — Сервис предиктивной аналитики («Вероятность завала СОЧ») и генерации отчетов.
* `scheduler.py` — Алгоритм автоматической генерации расписания и логика «умных замен» при болезни учителя.
* `main.py` — Точка входа, эмулятор API BilimClass и основные эндпоинты.

## 🚀 Быстрый старт

### 1. Установка зависимостей
```bash
pip install fastapi uvicorn sqlalchemy pydantic psycopg2-binary
```

### 2. Запуск сервера
```bash
uvicorn main:app --reload
```
После запуска документация Swagger будет доступна по адресу: `http://127.0.0.1:8000/docs`

## 📡 Основные модули API

### 📺 Kiosk Mode (Интерактивная стенгазета)
`GET /api/kiosk/wall-newspaper`  
Возвращает данные для школьных киосков: топ-учеников дня, актуальные замены в расписании и анонсы мероприятий.

### 🤖 AI Analytics
`GET /api/analytics/predict/{student_id}`  
Вычисляет вероятность неуспешной сдачи СОЧ на основе тренда последних оценок и посещаемости.

### 🗓 Smart Scheduler & Triggers
`POST /api/admin/teacher-sick/{teacher_id}`  
**Триггер:** Автоматически помечает учителя как неактивного, ищет замену среди свободных учителей с подходящей квалификацией и обновляет расписание, генерируя список уведомлений.

### 🧪 Mock BilimClass API
`GET /api/mock/bilimclass/grades/{student_id}`  
Эмулятор внешней системы. Используется для тестирования интеграции и получения случайных оценок учеников.

## 📊 Логика расчета рейтинга
В системе реализован авторский алгоритм расчета рейтинга (в `ai_service.py`), который учитывает:
1. **Academic Score (70%)** — средневзвешенный балл.
2. **Attendance (30%)** — процент посещаемости занятий.
